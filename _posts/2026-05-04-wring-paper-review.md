> **요약:** ICLR 2026에 채택된 MIT·WPI·Google의 논문 "WRING Out The Bias"는 Vision-Language Model(VLM)의 편향 제거 문제에서 기존 방식인 **Projection Debiasing**이 야기하는 '두더지 잡기 딜레마'를 해결합니다. 핵심 아이디어는 편향 서브스페이스를 **제거**하는 대신 **회전**시켜, 목표 편향은 제거하되 다른 관계는 최소한으로 건드리는 것입니다. 재학습 없이 사전학습 모델에 바로 적용할 수 있는 포스트프로세싱 방식입니다.

---

병원에서 피부 병변을 AI로 분류하는 모델이 있다고 합시다. 이 모델이 특정 피부 톤에 편향되어 있다면, 실제로 고위험 환자를 놓칠 수 있습니다. AI 공정성 문제는 단순히 윤리적 의무를 넘어, 생명과 직결된 안전 문제입니다.

이를 해결하기 위해 VLM의 편향을 제거하려는 연구가 이어지고 있지만, 지금까지의 접근 방식에는 치명적인 부작용이 있었습니다. 성별 편향을 고쳤더니 인종 편향이 강해지는 것처럼, **하나를 고치면 다른 곳이 터지는** 현상이 반복됐습니다. 마치 두더지 잡기 게임처럼요.

MIT, Worcester Polytechnic Institute, Google의 연구팀이 ICLR 2026에 발표한 **WRING(Weighted Rotational DebiasING)**은 이 문제를 수학적으로 다른 방식으로 접근해 해결합니다.

---

## 1. 배경: VLM의 편향이란 무엇인가?

### VLM과 CLIP

**Vision-Language Model(VLM)**은 이미지와 텍스트를 동시에 이해하는 멀티모달 모델입니다. OpenAI의 **CLIP(Contrastive Language-Image Pre-training)**이 대표적으로, 이미지와 텍스트를 같은 고차원 임베딩 공간에 매핑해 의미적 유사도를 계산합니다.

```
텍스트: "충성스러운 개"
이미지: 강아지 사진들
→ CLIP은 두 modality를 같은 벡터 공간에서 연결
```

CLIP은 의료 영상 분류, 이미지 검색, 의류 추천 등 다양한 분야에서 활용됩니다. 하지만 방대한 인터넷 데이터로 학습된 만큼, 그 데이터 속에 녹아든 사회적 편향도 함께 학습합니다.

### VLM의 편향: 허위 상관관계

논문에서 다루는 핵심 문제는 **Spurious Correlation(허위 상관관계)**입니다. 모델이 실제로 의미 있는 특징이 아닌, 학습 데이터에서 우연히 함께 등장했던 배경 속성과 라벨을 연결짓는 것입니다.

논문에서 제시한 실제 예시를 보면 명확합니다.

```
"충성스러운 개(loyal dogs)" 쿼리 → CLIP이 코기(Corgi)만 검색
→ 코기가 충성스럽다는 연관이 학습 데이터에 과도하게 존재했기 때문

"비싼 물건(expensive items)" 쿼리 → CLIP이 흑백 이미지만 검색
→ 흑백 이미지가 고급품과 연관된 학습 데이터 패턴을 반영
```

더 심각한 것은 의료 분야의 사례입니다. 피부 병변 분류 모델이 특정 피부 톤을 가진 환자에게 더 낮은 정확도를 보이거나, 병원 직원 이미지 검색 시 특정 인종이나 성별로 편향된 결과를 내놓는 문제가 실제로 보고되고 있습니다.

## 2. 기존 접근: Projection Debiasing와 그 한계

### Projection Debiasing란?

VLM 편향 제거의 가장 일반적인 접근 방식입니다. 임베딩 공간에서 편향을 유발하는 서브스페이스를 찾아낸 뒤, 그 서브스페이스를 **완전히 제거(투영)**하는 방식입니다.

수식으로 표현하면, 편향 벡터 방향 $\mathbf{v}$에 대한 투영 행렬 $P$를 이용해 임베딩 $\mathbf{e}$를 변환합니다.

$$\mathbf{e}' = \mathbf{e} - (\mathbf{e} \cdot \mathbf{v})\mathbf{v} = P\mathbf{e}$$

직관적으로 설명하면, 임베딩 공간에서 "편향과 관련된 방향"을 통째로 잘라내는 것입니다.

```
[임베딩 공간]
            편향 방향 (인종)
                ↑
────────────────┼──────────────── (다른 관계들)
                │
           이 방향을 통째로 제거
```

### Whac-A-Mole 딜레마 — 하나 잡으면 다른 게 나온다

그런데 Projection Debiasing에는 근본적인 부작용이 있습니다. Walter Gerych(논문 제1저자, 현 WPI 조교수)는 이렇게 설명합니다.

> "그렇게 하면, 주변의 모든 것이 같이 찌부러집니다. 모델이 학습한 다른 관계들도 그 서브스페이스를 제거할 때 함께 변화합니다."

이를 2023년 AI 연구에서 공식화한 **"Whac-A-Mole 딜레마"**라고 합니다. 현실에서 나타나는 패턴은 이렇습니다.

```
[Projection Debiasing 적용 시]

인종 편향 제거 → 성별 편향 증폭 😨
성별 편향 제거 → 연령 편향 증폭 😨
연령 편향 제거 → 인종 편향 재등장 😨

마치 두더지 잡기 게임처럼 하나를 누르면 다른 곳이 나옴
```

Marzyeh Ghassemi(MIT 전기공학·컴퓨터과학 부교수, 논문 공저자)는 이 문제의 실질적 위험성을 이렇게 설명합니다.

> "병원 직원 이미지를 검색하는 VLM에서 인종 편향을 제거하면, 의도치 않게 성별 편향을 증폭시키는 결과를 낳을 수 있습니다."

### 왜 이런 일이 생기는가?

수학적으로 보면, 고차원 임베딩 공간에서 한 방향을 완전히 제거하면 그 공간 전체의 기하학적 구조가 왜곡됩니다. 편향들 사이에는 우리가 명시하지 않은 미묘한 상관관계가 존재하는데, 하나를 잘라내면 다른 방향들이 이 공백을 채우며 재정렬됩니다.

더 근본적인 문제가 있습니다. **세상의 모든 편향을 미리 나열하는 것은 불가능합니다.** 우리가 제거하려는 편향 목록에 없는 편향이 증폭되어도, 평가 단계에서 그것을 측정조차 못할 수 있습니다.

## 3. WRING: 제거 대신 회전

### 핵심 직관

WRING의 핵심 아이디어는 한 문장으로 요약됩니다.

> **"편향이 있는 서브스페이스를 제거하지 말고, 그 안에서만 회전시켜라."**

편향을 담고 있는 좌표들을 다른 각도로 움직여서 모델이 특정 개념 내에서 서로 다른 그룹을 구분하지 못하게 만드는 것입니다. 이렇게 하면 **그 특정 서브스페이스 내에서만 표현이 바뀌고**, 모델의 다른 관계들은 그대로 유지됩니다.

비유로 설명하면 이렇습니다.

```
[Projection Debiasing — 벽 허물기]
편향을 담은 공간을 통째로 벽을 허물어 제거
→ 주변 공간 전체가 무너짐 (부수 피해 발생)

[WRING — 방향 바꾸기]
그 공간 안에서만 방향을 돌려 편향 정보가 희석되게 함
→ 공간은 그대로, 특정 정보만 정밀하게 조정
```

### 수학적 원리: 회전 행렬

Projection Debiasing가 서브스페이스를 제거하는 **투영 행렬(Projection Matrix)** $P$를 사용하는 반면, WRING은 서브스페이스 내에서만 벡터를 회전시키는 **회전 행렬(Rotation Matrix)** $R$을 사용합니다.

$$\mathbf{e}' = R\mathbf{e}$$

여기서 핵심은 $R$이 **직교 행렬(Orthogonal Matrix)**, 즉 $R^TR = I$를 만족한다는 점입니다. 이 성질이 중요한 이유가 있습니다.

투영 행렬 $P$는 임베딩의 노름(norm, 크기)을 줄입니다. 정보를 물리적으로 제거하기 때문입니다. 반면 직교 회전 행렬 $R$은 노름을 **보존**합니다. 정보를 없애는 것이 아니라, 방향만 바꾸는 것이기 때문입니다.

$$\|\mathbf{e}'\| = \|R\mathbf{e}\| = \|\mathbf{e}\|$$

"Weighted"라는 이름은 모든 편향 방향에 동일한 비중을 두지 않고, 편향 기여도에 따라 회전 각도를 다르게 적용하기 때문에 붙었습니다. 편향 기여가 큰 방향은 더 크게 회전하고, 작은 방향은 더 조금 회전합니다.

### 적용 범위

WRING은 **포스트프로세싱(Post-processing)** 방식입니다. 사전학습된 VLM에 추가 학습 없이 바로 적용할 수 있습니다.

Gerych는 이 점의 중요성을 이렇게 설명합니다.

> "사람들은 이미 엄청난 자원과 비용을 들여 이 거대한 모델들을 학습했습니다. 학습 과정에서 무언가를 수정하면 처음부터 다시 시작해야 하기 때문에, 그렇게 하고 싶지 않습니다. WRING은 매우 효율적이고, 모델 재학습이 필요 없으며, 최소한으로만 개입합니다."

현재 WRING은 CLIP 계열 모델에 주로 적용 가능합니다. 이미지와 언어를 연결하는 검색이나 분류에 쓰이는 CLIP 유형의 VLM이 주된 대상입니다.

## 4. Projection vs WRING: 직접 비교

### 기하학적 비교

```
[고차원 임베딩 공간 내 두 방법의 차이]

편향 서브스페이스: ███ (제거 또는 회전 대상)
기타 관계:        ... (보존 목표)

Projection Debiasing:
  ███ → (제거됨)
  ... → ...' (주변 공간이 왜곡되어 변화)
  
  문제: 제거 과정에서 주변 공간 전체가 재정렬됨

WRING:
  ███ → ▓▓▓ (서브스페이스 내에서 회전)
  ... → ... (주변 공간은 거의 그대로)
  
  효과: 목표 편향만 정밀하게 조정
```

### 핵심 차이 요약

| 항목 | Projection Debiasing | **WRING** |
| :--- | :--- | :--- |
| **접근 방식** | 편향 서브스페이스 **제거** | 편향 서브스페이스 내 **회전** |
| **수학적 도구** | 투영 행렬 $P$ | 직교 회전 행렬 $R$ |
| **노름 보존** | ✗ (정보 손실 발생) | ✅ (크기 보존) |
| **비대상 관계 영향** | 크다 (Whac-A-Mole) | **최소화** |
| **재학습 필요** | 불필요 | **불필요** |
| **적용 방식** | 포스트프로세싱 | **포스트프로세싱** |
| **알려지지 않은 편향 증폭** | 가능성 높음 | **가능성 낮음** |

## 5. 실험 결과

### WRING 적용 전후: 구체적 사례

논문에서 제시한 사례를 보면 WRING의 효과가 직관적으로 드러납니다.

**WRING 적용 전:**

```
"충성스러운 개(loyal dogs)" 검색 시
→ 코기(Corgi) 이미지만 집중적으로 반환
  ✗ 학습 데이터의 허위 상관관계가 그대로 반영

"비싼 물건(expensive items)" 검색 시
→ 흑백 이미지만 집중적으로 반환
  ✗ 흑백 = 고급품이라는 허위 연관이 반영
```

**WRING 적용 후:**

```
"충성스러운 개(loyal dogs)" 검색 시
→ 다양한 품종의 개들이 고르게 반환
  ✅ 코기에 대한 편향이 제거됨

"비싼 물건(expensive items)" 검색 시
→ 컬러/흑백 무관하게 다양한 고가품 반환
  ✅ 흑백 편향이 제거됨
```

### 주요 실험 결과

논문의 실험에서 연구팀은 WRING을 OpenCLIP에 적용해 다음을 검증했습니다.

**목표 편향 제거 효과:**
연구팀은 WRING이 목표 개념에 대한 편향을 유의미하게 감소시키면서도 다른 영역에서의 편향을 증가시키지 않는다는 것을 실험 결과로 확인했습니다.

**Whac-A-Mole 딜레마 극복:**
Projection Debiasing는 목표 편향을 제거하는 데 성공하지만 비대상 개념에서의 편향이 증폭되는 문제가 관찰된 반면, WRING은 목표 편향 제거와 비대상 편향 보존을 동시에 달성했습니다.

**의료 시나리오 검증:**
병원 직원 이미지 검색 태스크에서 인종 편향 제거 시 성별 편향이 증폭되는 Projection Debiasing의 부작용이 WRING에서는 발생하지 않았습니다. 이는 의료 AI 공정성 측면에서 특히 중요한 결과입니다.

## 6. 왜 이 논문이 중요한가?

### 1. 편향 제거의 근본 문제를 다룬다

기존 연구들의 대부분은 "어떤 편향을 제거할 것인가"에 집중했습니다. WRING은 한 발 물러서서 **"편향을 제거하는 방법 자체가 새로운 문제를 만들지는 않는가"**라는 메타 질문을 던집니다. 이것이 연구의 핵심 기여입니다.

### 2. 열거 불가능성(Unenumerability) 문제 인식

현실에서 모든 편향을 미리 파악해 열거하는 것은 불가능합니다. 논문이 강조하는 핵심 인사이트는 이것입니다.

> "실제로는 모든 가능한 편향을 열거하는 것이 불가능합니다. 즉, 편향의 증가가 평가 단계에서 관찰되지 않은 채로 지나갈 수 있습니다."

WRING은 이 현실적 제약을 전제로 설계됩니다. 알려진 편향을 제거하면서, 알려지지 않은 편향에 미치는 영향을 최소화하는 것이 목표입니다.

### 3. 실용적 배포 가능성

모델 재학습 없이 포스트프로세싱으로 적용 가능하다는 점은 실제 프로덕션 환경에서 매우 중요합니다. 수십억 파라미터 규모의 모델을 처음부터 재학습하는 것은 현실적으로 불가능하거나 극도로 비용이 큽니다. WRING은 이미 배포된 모델에도 바로 적용할 수 있습니다.

### 4. 의료 AI의 안전성 향상

이 연구는 MIT Abdul Latif Jameel 의료 머신러닝 클리닉의 지원 하에 이루어졌습니다. 피부 병변 분류, 병원 직원 검색 등 의료 분야에서 편향이 초래하는 실질적 피해를 줄이는 것이 궁극적 목표입니다. 의료 AI에서 편향은 단순한 성능 문제가 아니라 환자 안전과 직결된 사안이기 때문입니다.

## 7. 한계와 향후 연구 방향

### 현재 한계

**CLIP 계열 모델에 한정:**
현재 WRING은 CLIP 타입의 VLM, 즉 이미지와 텍스트를 대조적으로 학습한 모델에 주로 적용됩니다. ChatGPT 스타일의 생성형 언어 모델에 확장하는 것이 다음 과제입니다.

Gerych는 이 점을 명확히 밝힙니다.

> "ChatGPT 스타일의 생성형 언어 모델로 확장하는 것이 우리의 합리적인 다음 단계입니다."

**알려진 편향에 한정:**
WRING도 어디까지나 연구자가 명시한 편향을 대상으로 작동합니다. 우리가 인식조차 하지 못하는 편향은 여전히 다룰 수 없습니다.

### 향후 연구 방향

**생성형 VLM으로의 확장:**
CLIP-BLIP, LLaVA, GPT-4o 같은 생성형 VLM은 단순 분류·검색을 넘어 텍스트를 생성합니다. 생성 과정에서의 편향은 더 복잡한 문제를 야기하므로, WRING의 원리를 이 영역으로 확장하는 연구가 필요합니다.

**자동 편향 탐지와 연동:**
WRING이 제거할 편향을 어떻게 식별할지에 대한 연구와 결합하면 더 완전한 파이프라인이 될 수 있습니다. 알려지지 않은 편향을 자동으로 탐지해 WRING의 입력으로 제공하는 시스템을 상상할 수 있습니다.

**의료 도메인 특화:**
의료 이미징, 임상 기록 등 특수 도메인에서의 편향은 일반 데이터의 편향과 다른 특성을 가집니다. 도메인 특화 WRING 변형 연구가 기대됩니다.

| **적용 방식** | 포스트프로세싱, 재학습 불필요 |

| **한계** | 생성형 LLM 미지원, 알려진 편향만 처리 |

WRING이 말하는 핵심 메시지는 단순하지만 강력합니다. **편향을 제거하는 방식이 새로운 편향을 만들어서는 안 된다.** 이를 위해 "없애는" 대신 "돌리는" 전략을 택했고, 그 수학적 근거로 직교 행렬의 노름 보존 성질을 활용했습니다.

AI 공정성 연구에서 "한 편향을 고치면 다른 편향이 나온다"는 딜레마는 오래된 골칫거리였습니다. WRING은 이 문제에 수학적으로 우아한 해결책을 제시하며, 특히 의료 AI처럼 공정성이 생명과 직결되는 영역에서 실질적인 기여를 할 수 있는 연구입니다.
